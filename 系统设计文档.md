2. 系统总体架构
本项目采用模块化工程结构，确保代码的可维护性与扩展性：
2.1 硬件总体框图
主控： STM32F103C8T6 核心板。

通信总线： I2C 总线复用，连接 OLED 屏幕与 MPU6050 传感器，布线简洁，降低系统干扰。

无线接口： 蓝牙模块通过 UART 通信，作为外部数据交互的核心通道。

2.2 
硬件驱动层： 基于 STM32CubeMX 配置的 I2C、UART 及定时器驱动。

OS 中间件层（进行中）： 引入 FreeRTOS 实时内核，划分 UI 显示、传感器采集、蓝牙同步及计时四个任务。

应用逻辑层： 处理菜单逻辑、姿态解算及计步算法。

3.详细设计文档

3.1 已实现模块功能说明
蓝牙通信模块（Phase3）： 利用 UART 串口中断接收机制，实现了上位机指令的实时解析与手表状态的回传。针对通信协议进行了校验位设计，确保了长连接下的数据稳定性。

OLED 显示与传感器采集（Phase1/2）： 已跑通 I2C 驱动，能够实时读取 MPU6050 三轴数据并进行 OLED 显示。

3.2 资源冲突与优化计划
针对多设备在 I2C 总线上的潜在冲突，我们将利用 FreeRTOS 的 互斥信号量（Mutex） 机制进行管理，确保 Sensor_Task 和 UI_Task 在访问总线时互斥访问。
